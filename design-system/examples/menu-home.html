schema Anatomy

1

Top Bar

App logo, language toggle (EN/JP), and user avatar — persistent across all authenticated views.

2

Onboarding Card

Dismissible info card explaining core features — shown only on first visit (State 1), hidden via close button.

3

Published Menus (Dual-Slot Grid)

Two-column grid with Primary and Secondary slots. Each slot shows either an empty card with "Create Menu" CTA or a published card with QR code, actions, and overflow menu (Edit, Replace, Unpublish).

4

Previous Menus

Version history list with View, Publish (opens slot picker modal), and Delete actions per item. Appears when menus have been unpublished or replaced.

5

Publish Modal

Slot picker dialog for publishing from Previous Menus. Shows occupied slot warnings when a slot already has a published menu.

layers Page States

State 1 — Empty

First visit. No menus published, no previous menus. Onboarding card visible. Both slots show empty cards with "Create Menu" CTA.

State 2 — Primary Published

Primary slot occupied with published card (QR code, guest view, copy link, download QR). Secondary slot empty. No previous menus yet.

State 3 — Both Published

Both slots occupied. Previous menus list appears with View, Publish (slot picker modal with warnings), and Delete actions.

State 4 — After Unpublish

Primary unpublished and moved to top of Previous Menus. Both slots empty again. No onboarding card (returning user).
