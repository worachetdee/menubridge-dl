schema Anatomy

1

Restaurant Header

Restaurant name and subtitle displayed at the top — sets context for the guest.

2

Dietary Filter Pills

Horizontal filter bar (All, Vegetarian, Fish, Meat) — guests tap to filter items by dietary category. Active pill is filled dark.

3

Search Bar

Full-width search input for quick item lookup by name or ingredient keyword.

4

Guest Notes

Subtle info box with restaurant policies — table charge, last order times, and other notices set by the restaurant owner.

5

Menu Sections

Grouped by category (ON TAP, とりあえず一品, 揚げ物・焼き物). Each section has a bold header with optional subtitle notes.

6

Menu Item Row

Each item shows Japanese name, parenthetical description, price(s), and a chevron indicating a detail view is available. ON TAP items show half/pint pricing.

code Design Notes

  • Mobile-first: This is the primary view guests see via QR code on their phones. The layout is optimized for single-column mobile reading.
  • Bilingual Content: Item names are in Japanese with descriptions in parentheses. The EN/JP toggle on the owner side controls which translations are shown.
  • Filter Pills: Dietary filters use pill buttons with filled/outline states. "All" is the default active state.
  • Chevron Detail: Each item row has a right chevron indicating tappable detail — leads to ingredient breakdown, dietary tags, and guest explanation.
  • Minimal Chrome: No app navigation bar — this is a public-facing page accessed via shared link or QR code. Clean, distraction-free reading experience.
  • Themeable: The menu supports 4 themes via CSS custom properties — Craft Brewery (Inter, clean), Fine Dining (Playfair Display, dark/gold), Izakaya (Zen Maru Gothic, warm earth tones), and Modern Cafe (DM Serif, natural sage green). Themes control fonts, colors, border radius, and section title treatment.
